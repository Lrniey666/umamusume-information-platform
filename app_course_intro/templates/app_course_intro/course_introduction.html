1
資料層 · Phase 1
🕷 新聞爬蟲（5 個來源）
定期爬取賽馬娘相關公告與新聞，儲存至 GameAnnouncement 資料表。 支援排程自動執行，覆蓋繁中主要媒體平台。
📢 Bilibili BWIKI 🎮 巴哈姆特哈啦板 📰 聯合新聞網 📰 ETtoday 📰 宅宅新聞
requests / BeautifulSoup APScheduler Django ORM
2
分析層 · Phase 2
📊 情緒分析管線
雙軌情緒分析：使用 Gemini API 對公告進行三分類標記（正/中/負）， 使用 SnowNLP 對 Bilibili 評論進行快速情緒評分。 分析結果存入資料庫，供儀表板視覺化呈現。
Gemini API SnowNLP 正面 / 中立 / 負面 Chart.js 儀表板
3
AI Agent 層 · Phase 3a
🧠 AI Agent — 基礎對話 (app_agent_uma)
第一階段 Agent：單輪 LLM 呼叫 + DB 查詢。 使用者輸入問題，後端查詢 GameAnnouncement 並注入 Prompt， 由 Gemini / Claude 生成回答。不具備 Function Calling 能力。
Gemini 2.5 Flash System Prompt 設計 Session 對話歷史
4
RAG 層 · Phase 3b
🔍 RAG 知識庫問答 (app_rag_uma)
建立 FAISS 向量索引：將所有公告文字以 Gemini Embedding 模型轉為向量， 儲存為 uma_knowledge.faiss 與 uma_knowledge_texts.pkl。 使用者提問時，先做語意搜尋取回最相關段落，再注入 Prompt 讓 LLM 回答。
FAISS gemini-embedding-001 Top-K 語意搜尋 RAG Pipeline
5
RAG 層 · Phase 3c
🤖 Agentic RAG (app_rag_agent)
進階架構：LLM 具備 Function Calling 能力，可動態決定呼叫哪個工具。 工具一：search_uma_announcements（語意搜尋 FAISS + DB 模糊比對）。 工具二：list_announcements_by_category（精確分類查詢）。 透過多輪 Agentic 迴圈完成複雜查詢，並引用來源。
Function Calling Agentic Loop FAISS 語意搜尋 DB 精確查詢 marked.js 渲染
6
AI Agent 層 · Phase 4a
⛓ LangChain Agent
以 LangChain 框架重構 Agent：使用 AgentExecutor + Tool 定義工具， 支援 ReAct 推理模式（思考 → 行動 → 觀察 → 再思考）。 整合 LangChain Memory 管理多輪對話，支援 Streaming 輸出。
LangChain ReAct Agent AgentExecutor Streaming 輸出 ConversationMemory
7
AI Agent 層 · Phase 4b
🕸 LangGraph Agent
以有向圖（DAG）定義 Agent 工作流：節點（Node）代表步驟， 邊（Edge）代表條件路由。支援條件分支、回環、平行執行節點。 比 LangChain AgentExecutor 更靈活，適合複雜多工 Agent 設計。
LangGraph StateGraph 條件路由 Edge 平行節點 Checkpointing
8
整合層 · Phase 5a
💬 Discord Bot 整合
Discord Bot 作為前端互動介面：透過 Slash 指令觸發 RAG 問答、 公告查詢、情緒分析報告。同時可在指定頻道自動推播最新爬取公告， 嵌入卡片展示角色情報。
discord.py Slash Commands Embed Cards RAG API 呼叫 自動推播
9
整合層 · Phase 5b
▶ YouTube 整合
透過 YouTube Data API v3 搜尋賽馬娘相關影片， 抓取留言進行情緒分析，並將熱門影片資訊嵌入 Discord 頻道。 支援關鍵詞觀看聲量趨勢分析。
YouTube Data API v3 search.list commentThreads.list SnowNLP 評論情緒 Discord 嵌入
